# 気温くらべ帳 (Weather Almanac)

カレンダーから日付をひとつ選ぶと、今年から過去5年・過去10年分の気温を比較できるシングルページのWebアプリです。ビルド不要・外部ライブラリ不要の1枚のHTMLファイルで動きます。

## デモ

GitHub Pagesで公開すると以下のようなURLになります。

```
https://(あなたのユーザー名).github.io/(リポジトリ名)/
```

## 主な機能

- **カレンダー選択**: 今年のカレンダーから日付をタップして選択
- **日ごと / 週平均 / 月平均**: タブで比較の粒度を切り替え
  - 日ごと: 選んだ日そのものの気温
  - 週平均: 選んだ日を含む週(日〜土)の平均
  - 月平均: 選んだ日が属する月全体の平均
- **比較年数の切り替え**: 過去5年 / 過去10年
- **平年差**: 表示中の年数の平均気温を基準に、各年が何度高い/低いかを色付きで表示
- **降水量・湿度**: 各年の合計降水量(mm)と平均湿度(%)も表示
- **気温の推移グラフ**: 最高・平均・最低気温を年ごとの折れ線グラフで表示
- **地点の指定方法**:
  - ブラウザの現在地取得(位置情報の許可が必要)
  - 主要都市ボタン(東京・名古屋・大阪・札幌・福岡・那覇)
  - 緯度・経度の直接入力

## 使っているデータ

気温・降水量・湿度のデータは [Open-Meteo](https://open-meteo.com/)(CC BY 4.0)から取得しています。APIキーは不要です。

- 直近1週間程度: Forecast API(速報値)
- それ以前: Historical Weather (Archive) API(再解析データ)

現在地の地名表示には [BigDataCloud](https://www.bigdatacloud.com/) の逆ジオコーディングAPIを使用しています。

## 技術構成

- 素のHTML / CSS / JavaScript(フレームワーク・ビルドツール不要)

## 使い方(公開の仕方)

このアプリは通常のWebページとして動かす必要があります(位置情報やAPI通信を使うため)。

### GitHub Pagesで公開する場合

1. GitHubで新しいリポジトリを作成(Public)
2. このファイル(`index.html`)をアップロード
   - ファイル名は必ず `index.html` にしてください
3. リポジトリの `Settings` → `Pages` を開く
4. `Build and deployment` の `Source` を `Deploy from a branch` に設定
5. `Branch` で `main` / `/ (root)` を選択して `Save`
6. 数分待つと `https://(ユーザー名).github.io/(リポジトリ名)/` で公開されます

### ローカルで確認する場合

`index.html` をブラウザで直接開くか、簡易サーバーを立てて確認できます。

```bash
python3 -m http.server 8000
```

その後ブラウザで `http://localhost:8000` を開いてください。

## 注意事項

- 無料ホスティングサービスによっては、サーバー側の設定で位置情報(Geolocation)がブロックされる場合があります。その場合は都市ボタンか緯度・経度の直接入力をご利用ください。
- Open-Meteoは非商用利用で1日あたり10,000リクエストまで無料です。個人利用の範囲では問題になりません。
- データの著作権表示: データ提供元は Open-Meteo.com (CC BY 4.0) です。
